# PoshMcp v0.9.11 Release Notes

## What's New

### Features

- **OAuth: Add /authorize proxy redirect endpoint** — The GET `/authorize` endpoint now acts as a proxy that redirects to Entra's authorize URL with full OAuth2 PKCE parameter support. VS Code MCP client was constructing the auth URL as `{proxy_base}/authorize` and getting a 404. This endpoint accepts all OAuth2 PKCE params and issues a 302 redirect to `login.microsoftonline.com/{tenantId}/oauth2/v2.0/authorize`, replacing the ephemeral DCR client_id with the real Entra client_id from config.

## Bug Fixes

- **OAuth flow now completes for VS Code MCP clients** — The `/authorize` redirect endpoint resolves the issue where VS Code clients were unable to reach Entra's authorization endpoint, causing OAuth authentication to fail silently.

## Upgrade Notes

No breaking changes. Update your configuration to ensure `Authentication.ClientId` and `Authentication.TenantId` are properly set in your environment or `appsettings.json`.
