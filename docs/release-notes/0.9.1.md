# PoshMcp v0.9.1 Release Notes

**Released**: 2026-05-01

## What's New

### Enhanced Diagnostics: Authentication and Identity Sections

The `poshmcp doctor` command and the `get-configuration-troubleshooting` MCP tool now provide deeper insight into authentication configuration and request-scoped identity:

- **Authentication section** — Configuration-based diagnostics showing:
  - Enabled state and configured authentication schemes (type: JwtBearer or ApiKey)
  - Authority and audience URIs for OAuth
  - Key count for API key schemes
  - Default policy (required scopes, required roles)
  - Protected resource URI and CORS origins
  - **No secrets are ever exposed** — keys and credentials are redacted
  
- **Identity section** — Request-scoped caller identity (HTTP transport only):
  - Authenticated state and authentication scheme name
  - Principal name (subject claim)
  - Assigned scopes (from `scp` claims) and roles
  - Shows `available: false` in CLI doctor and stdio transport modes
  - Only populated when an HTTP context is present

### Dead Code Removal

- Removed unused `ConfigurationTroubleshootingTools.cs` class that was never instantiated
- Cleaner codebase with no functional impact

## Breaking Changes

None. The new diagnostic sections are additive; all existing API behavior is unchanged.

## Upgrade Notes

### For Existing Deployments

- No configuration changes required
- Run `poshmcp doctor` to see the new authentication and identity diagnostics
- If using HTTP transport, identity information will now be included in the output
- CLI and stdio mode deployments will show `available: false` for the identity section (expected)

### For Troubleshooting

Use the expanded doctor output to:
- Verify authentication configuration is loaded correctly
- Check that all configured scopes and roles are present
- View the request-scoped caller identity when deployed in HTTP mode
- Ensure no secrets are leaked in logging

## Testing

This release includes:
- 7 new unit tests covering authentication section population from `AuthenticationConfiguration`
- Tests verifying identity section population from `ClaimsPrincipal` in HTTP context
- Validation tests ensuring no secrets are exposed in diagnostics
- All 403 unit tests pass

## Issues Closed

- Enhanced `poshmcp doctor` with authentication diagnostics
- Enhanced `get-configuration-troubleshooting` MCP tool with identity section
- Removed dead code (`ConfigurationTroubleshootingTools.cs`)
