# PoshMcp v0.9.8 Release Notes

**Released**: 2026-05-02

## Overview

This release fixes a regression introduced in v0.9.7 where `IOptions<AuthenticationConfiguration>`
was no longer resolvable when `AddPoshMcpAuthentication` was called without a preceding
`AddOptions` registration (e.g., in tests and the `CommandHandlers` CLI helper). It also
makes several `Program` helper methods `internal` so the extracted `CommandHandlers` class
can call them without reflection.

## What's Fixed

### `IOptions<AuthenticationConfiguration>` Not Registered by `AddPoshMcpAuthentication`

**Symptom**: Three unit tests in `AuthenticationServiceExtensionsTests` failed because
`IOptions<AuthenticationConfiguration>` could not be resolved from the DI container when
only `AddPoshMcpAuthentication` was called:

- `WhenAuthEnabled_IOptionsAuthenticationConfiguration_ReflectsConfig`
- `WhenAuthDisabled_IOptionsAuthenticationConfiguration_IsRegisteredWithEnabledFalse`
- `WhenNoAuthSection_IOptionsAuthenticationConfiguration_DoesNotThrow`

**Root Cause**: v0.9.7 removed the `services.Configure<AuthenticationConfiguration>` call
from `AddPoshMcpAuthentication` to eliminate duplicate registrations. However, the duplicate
was actually the call in `RunHttpTransportServerAsync` (Program.cs), not the one inside
`AddPoshMcpAuthentication`. Removing the wrong registration left `AddPoshMcpAuthentication`
without any `IOptions` registration of its own.

**Fix**:
- Moved `AddOptions<AuthenticationConfiguration>().Configure(...).ValidateOnStart()` into
  `AddPoshMcpAuthentication` so the registration is self-contained.
- Removed the now-redundant `AddOptions` block from `RunHttpTransportServerAsync` in
  `Program.cs`, which was the actual duplicate.

### `Program` Helper Methods Made `internal`

Several `Program` static methods used by the newly extracted `CommandHandlers` class were
`private`, causing compiler errors. Changed to `internal`:

- `DescribeConfigurationPath`
- `AddConfigurationGuidanceToolToList`
- `AddConfigurationTroubleshootingToolToList`
- `CreateToolFactory`
- `StartOutOfProcessExecutorIfNeededAsync`
- `OutOfProcessExecutorLease` (nested class)

## Upgrade Notes

No configuration changes required. Behavior is identical to v0.9.7 — this release only
restores the registration path and fixes internal access modifiers.

## Compatibility

- No breaking changes from v0.9.7
- All v0.9.5/v0.9.6/v0.9.7 features remain unchanged
