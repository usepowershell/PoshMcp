# PoshMcp v0.9.0 Release Notes

**Released**: 2026-04-29

## What's New

### Application Insights Integration (Milestone 4 — Spec 008)

PoshMcp now supports optional Azure Application Insights telemetry via OpenTelemetry, giving you full observability into your MCP server in production environments:

- **Opt-in telemetry** — Enable Application Insights by setting `ApplicationInsights.Enabled: true` in `appsettings.json` and providing a connection string. Disabled by default — no behavior change for existing users
- **Flexible connection string sourcing** — Provide the connection string via `appsettings.json` (`ApplicationInsights.ConnectionString`) or the `APPLICATIONINSIGHTS_CONNECTION_STRING` environment variable (env var takes precedence)
- **Configurable sampling** — Control telemetry volume with `ApplicationInsights.SamplingPercentage` (1–100), balancing observability with cost
- **Graceful degradation** — If the connection string is missing or invalid, the server starts normally and logs a warning. No crashes, no startup failures
- **OTel log suppression** — Internal OpenTelemetry diagnostic logs are suppressed to keep your log output clean

### Doctor Validation for Application Insights

The `poshmcp doctor` command now validates your Application Insights configuration:

- Validates connection string format (must start with `InstrumentationKey=`)
- Validates sampling percentage is within the 1–100 range
- No network calls — validation is purely structural, safe to run anywhere

## Configuration

Add the following to your `appsettings.json` to enable Application Insights:

```json
{
  "ApplicationInsights": {
    "Enabled": true,
    "ConnectionString": "InstrumentationKey=00000000-0000-0000-0000-000000000000;IngestionEndpoint=https://eastus-0.in.applicationinsights.azure.com/",
    "SamplingPercentage": 100
  }
}
```

Or use the environment variable:

```bash
export APPLICATIONINSIGHTS_CONNECTION_STRING="InstrumentationKey=..."
```

## Breaking Changes

None. Application Insights is disabled by default and all existing behavior is unchanged.

## Upgrade Notes

### For Existing Deployments

- No configuration changes required
- No breaking changes to API or CLI behavior
- To enable Application Insights, add the configuration block above
- For Docker deployments, pass the connection string as an environment variable

### For Azure Deployments

- Application Insights connection strings are available in the Azure Portal under your Application Insights resource → Overview → Connection String
- When using Managed Identity, the connection string is still required (it identifies the destination, not the authentication)

## Platform Support

- **Linux/macOS**: .NET 10 runtime
- **Docker**: Supported via environment variable injection
