# PoshMcp v0.8.3 Release Notes

**Released**: 2026-04-24

## What's New

### Deploy Script: Source Image Support

The `infrastructure/azure/deploy.ps1` script gains two new optional parameters that let you deploy a pre-built container image instead of always performing a local `docker build`.

**New parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `-SourceImage` | `string` | Container image reference to use in place of a local build (e.g. `ghcr.io/org/poshmcp:v1.2.0`) |
| `-UseRegistryCache` | `switch` | When combined with `-SourceImage`, imports the image directly into ACR via `az acr import` (no local pull) |

**Three execution modes**

- **Mode A — local pull** *(new)*: `-SourceImage` provided. The script pulls the source image locally, re-tags it for the target ACR, and pushes. No local build occurs.
- **Mode B — ACR import** *(new)*: `-SourceImage` and `-UseRegistryCache` both provided. Uses `az acr import` to copy the image into ACR server-side — ideal for large images or bandwidth-constrained environments.
- **Mode C — local build** *(unchanged)*: Default behavior when `-SourceImage` is omitted. No changes to existing workflows.

**Highlights**

- Eliminates 10–15-minute local builds when a known-good image already exists in a registry
- Enables artifact-promotion and one-image-many-deployments GitOps patterns
- Clear validation: `-UseRegistryCache` without `-SourceImage` exits immediately with an actionable error
- Backward-compatible — all existing parameters and environment variables are unchanged

### Code Quality Improvements

Internal code structure has been refactored for improved readability and maintainability. While this is not a user-facing feature change, it reduces the surface area for latent bugs and makes the codebase easier to extend in future releases.

## Breaking Changes

None. All existing CLI parameters, MCP tool schemas, and configuration formats are unchanged.

## Upgrade Notes

- Update to `0.8.3` via your preferred package source (`dotnet tool update`, NuGet, or container pull)
- No configuration migration is required
- Azure deployments using `deploy.ps1` continue to work without modification; the new `-SourceImage` and `-UseRegistryCache` parameters are fully optional
