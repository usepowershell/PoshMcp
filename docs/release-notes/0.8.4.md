# PoshMcp v0.8.4 Release Notes

**Released**: 2026-04-24

## Security Fixes

### CVE-2026-40894 — OpenTelemetry.Api Denial-of-Service (Moderate)

| Field | Detail |
|-------|--------|
| **CVE** | CVE-2026-40894 |
| **Severity** | Moderate |
| **Affected component** | `OpenTelemetry.Api` (upgraded from 1.15.1 → 1.15.3) |
| **Impact** | Unbounded memory allocation via `BaggagePropagator` and `B3Propagator` could allow a remote attacker to trigger a denial-of-service condition by sending crafted trace context headers |
| **Resolution** | Upgraded to `OpenTelemetry.Api` 1.15.3, which applies upstream mitigations for the allocation path |

No code changes or configuration updates are required. The fix is transparent to all existing deployments.

## Breaking Changes

None.

## Upgrade Notes

- Update to `0.8.4` via your preferred package source (`dotnet tool update`, NuGet, or container pull)
- No configuration migration is required
- The fix is applied automatically on upgrade; no changes to MCP tool schemas, configuration, or deployment scripts
