# PoshMcp v0.9.19 Release Notes

## Diagnostics

### Enhanced JWT claim logging at validation time

`OnTokenValidated` now emits two log lines per successful authentication:

1. **`JWT OnTokenValidated: AllClaims=[...]`** at Information level — every
   claim in the validated token, pipe-delimited.
2. **`JWT AUTHZ DIAG: aud=[...] scp=[...] roles=[...]`** at Warning level —
   isolates the three claims that matter for `McpAccess` policy authorization
   so they're easy to find in noisy log streams.

This makes it straightforward to diagnose 403s when authentication succeeds
but authorization fails: the warning line shows exactly what scopes and roles
the token carried, so operators can compare against `RequiredScopes` /
`RequiredRoles` in `appsettings.json` without decoding the JWT manually.
