# PoshMcp v0.8.0 Release Notes

**Released**: 2026-04-20

## What's New

### Docker Build Output Improvements

The `poshmcp build` command has been significantly improved with real-time output streaming and critical deadlock fixes:

- **Fixed stdout/stderr deadlock** — The Docker build process was hanging due to a pipe-buffer deadlock when reading output sequentially. This has been resolved by implementing concurrent task-based readers with `Task.WaitAll`, eliminating silent hangs that occurred even after Docker successfully built the image
- **Real-time build progress** — Build output now streams live to the console instead of buffering silently, giving immediate visibility into the build process
- **Refactored Docker integration** — `BuildDockerBuildArgs` has been extracted into `DockerRunner` as a testable static method, improving code quality and testability

### Enhanced Diagnostic Output

The `doctor` command has been significantly expanded to help diagnose configuration and deployment issues:

- **Authentication configuration** — Doctor output now displays all authentication settings for debugging identity and token issues
- **Logging configuration** — Logging setup is included in diagnostic output to verify logging is configured correctly
- **Environment variables** — All relevant environment variables are displayed for context when troubleshooting
- **MCP tool definitions** — MCP tool schemas and configurations are now visible in doctor output, helping users understand what tools are available and how they're configured
- **Hierarchical report structure** — Doctor output has been refactored to use a `DoctorReport` object hierarchy with a dedicated `DoctorTextRenderer`, providing cleaner separation of concerns and more structured diagnostic data

### Bug Fixes & Security

- **Dockerfile build fix** — Fixed `Dockerfile` to correctly build only the `PoshMcp.Server` project instead of the entire solution
- **Security dependency update** — Updated `System.Security.Cryptography.Xml` from 10.0.5 to 10.0.6 to address CVE vulnerabilities

## Breaking Changes

None. This is a feature and stability release.

## Upgrade Notes

### For Existing Deployments

- No configuration changes required
- No breaking changes to API or CLI behavior
- The Docker build hang fix is recommended for all users experiencing silent hangs during image builds
- New diagnostic capabilities in `doctor` command are available immediately

## Platform Support

- **Windows**: .NET 10 runtime
- **Linux/macOS**: .NET 10 runtime
- **Docker**: Included in `poshmcp:latest` image (no hanging on build)

## Testing

This release includes:
- 11 new unit tests in `DockerRunnerTests.cs` covering all Docker build scenarios
- Regression tests for the fixed stdout/stderr deadlock
- Tests for the new diagnostic output features in the `doctor` command
- Integration tests for the refactored `DockerRunner` class

## Highlights for Docker Users

If you've experienced the `poshmcp build` command hanging silently, this release fixes that issue. The concurrent reader implementation prevents pipe-buffer deadlocks, and real-time output streaming now gives you immediate feedback during builds.
