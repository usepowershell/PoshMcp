# PoshMcp v0.9.6 Release Notes

**Released**: 2026-05-02

## Overview

This release fixes a startup crash that prevented PoshMcp from running when authentication
environment variables were set but no configuration file was found at the expected path. In
env-only configuration mode (when `HasEnvironmentAppSettingsOverrides` returns true), the
out-of-process PowerShell executor received an empty string as the config file path, causing
`Path.GetFullPath("")` to throw `ArgumentException` before the server could start.

## What's Fixed

### Container Startup Crash with OutOfProcess Runtime Mode

**Symptom**: When deploying a PoshMcp container with `Authentication__*` or
`PowerShellConfiguration__*` environment variables set but no config file at a discoverable
path, the container would crash at startup with:

```
error: The value cannot be an empty string. (Parameter 'path')
```

The container revision would report `Failed` / `Unhealthy` health state and no requests
(including health checks) would be served.

**Root Cause**: `OutOfProcessCommandExecutor.SetupAsync` checked `configFilePath is not null`
before calling `Path.GetFullPath(configFilePath)`. When `HasEnvironmentAppSettingsOverrides()`
returns `true` (any `Authentication__*` or `PowerShellConfiguration__*` env var is set) and no
config file is found at the CWD or user profile path, `ResolveConfigurationPathWithSourceAsync`
returns an empty string (not null) as the config path. The empty string passed through the null
check and triggered `Path.GetFullPath("")` which throws `ArgumentException`.

**Fix**: Changed the guard to `!string.IsNullOrEmpty(configFilePath)` so empty strings fall
back to `Directory.GetCurrentDirectory()` (the same behavior as null).

**Impact**: Deployments using `POSHMCP_RUNTIME_MODE=OutOfProcess` with authentication or
PowerShell configuration environment variables set are now able to start successfully.

## Upgrade Notes

No configuration changes are required. If you were experiencing the startup crash described
above, upgrading to v0.9.6 will resolve it.

For containerized deployments using a bundled `appsettings.json`, the recommended approach is
to copy the file to the container's working directory (`/app/appsettings.json`). The server
discovers configuration files in the following order:

1. Path from `POSHMCP_CONFIGURATION` environment variable (if set)
2. `appsettings.json` in the current working directory (`/app/appsettings.json`)
3. User profile config (`~/.config/PoshMcp/appsettings.json` on Linux)
4. Environment variables only (if any `Authentication__*` / `PowerShellConfiguration__*` vars are set)
5. Embedded default config installed to user profile

This means that if your `Dockerfile` copies `appsettings.json` to `/app/appsettings.json`,
the full configuration (including `Authentication.Schemes`) is loaded from the file, and
individual settings can still be overridden via environment variables.

## Compatibility

- Requires .NET 10
- No breaking changes from v0.9.5
- All v0.9.5 features (OAuth AS proxy, DCR proxy, PRM auto-population) remain unchanged
