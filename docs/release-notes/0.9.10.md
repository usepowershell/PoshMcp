# PoshMcp v0.9.10 Release Notes

## What's New

### Bug Fixes

- **OAuth: Fix issuer mismatch in AS metadata** — The `/.well-known/oauth-authorization-server` response now returns the Entra ID issuer (`https://login.microsoftonline.com/{tenantId}/v2.0`) instead of the server's own URL. MCP clients that validate `token.iss == AS.issuer` (RFC 8414) were rejecting valid Entra tokens, causing an infinite `initialize` timeout loop.

## Upgrade Notes

No breaking changes. Update `appsettings.json` `RequiredScopes` to use the short scope name (`user_impersonation`) instead of the full URI form (`api://.../.../user_impersonation`) — Entra v2.0 tokens put `scp` as the short name only.
