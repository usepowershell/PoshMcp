## PowerShell Best Practices Applied

1. **Write-Host is anti-pattern** - Bypasses pipeline; use only for pure presentation
2. **Use appropriate streams:**
   - Write-Information: User-facing informational messages
   - Write-Verbose: Diagnostic details and execution traces
   - Write-Warning: Non-fatal warnings
   - Write-Error: Errors with proper categories
   - Write-Host: Final formatted output requiring color
3. **Error categories provide semantics** - NotInstalled, AuthenticationError, InvalidOperation
4. **CmdletBinding enables free parameters** - Automatic `-Verbose`, `-InformationAction`, etc.
5. **$InformationPreference for interactivity** - Set to 'Continue' for scripts run by humans
